# 설계 철학

Image to Editable PPT Skill이 해결하는 문제는 하나뿐입니다. “보이지만 수정할 수 없는” 슬라이드를 “객체를 하나씩 편집할 수 있는” PowerPoint로 되돌리는 것입니다.

이 skill의 설계와 조정에 관한 전체 경험은 다음 글에서 확인할 수 있습니다. [2000 个 GitHub Star 换来的经验：好的 AI Skill 是调出来的，不是写出来的](https://mp.weixin.qq.com/s/LaxWBX-nogHPpSxlk-Vs8Q)

## 페이지 전체를 이미지로 붙이지 않는 객체 단위 재구성

슬라이드 이미지 안의 내용은 편집 관점에서 가치가 서로 다릅니다. 이 skill은 각 페이지를 세 종류의 객체로 나누어 처리합니다.

- **읽을 수 있는 텍스트** → 가능한 한 네이티브 텍스트 상자로 복원해 글자, 크기, 색상을 직접 수정할 수 있게 합니다.
- **단순한 도형**(사각형, 원, 선, 화살표 등) → 가능한 한 PowerPoint 도형으로 복원해 이동, 크기 조정, 스타일 변경이 가능하게 합니다.
- **복잡한 시각 요소**(사진, 일러스트, 아이콘, 질감, 손그림 장식) → 출처가 기록된 독립 이미지 에셋으로 유지해 전체를 이동하거나 교체할 수 있게 하지만, 내부 편집 가능성은 보장하지 않습니다.

표, 차트, 순서도 등의 구조화 영역은 편집 가능한 의미 구조를 우선 보존합니다. 신뢰도가 낮으면 에셋으로 유지하고 검증 보고서에 설명합니다.

## 측정 기반 텍스트 복원

텍스트의 크기와 위치는 눈대중으로 정하지 않습니다. 변환이 시작되면 전체 입력을 하나의 일괄 작업으로 OCR(PaddleOCR-VL)에 제출해 각 페이지의 텍스트 주석을 생성합니다. 여기에는 정확한 상자 좌표, 원본 이미지의 실제 잉크 영역을 바탕으로 측정한 글자 크기, 같은 계층의 크기 그룹, 인식된 텍스트가 포함됩니다. AI는 재구성 시 이 측정값을 사용하며 같은 계층의 텍스트 크기를 자동으로 일관되게 유지합니다.

OCR Token을 설정하지 않으면 skill은 내장 오프라인 감지기로 폴백합니다. 이 감지기는 순수 기하 측정을 통해 텍스트의 위치와 크기는 알지만 내용을 인식하지 못하므로 텍스트 복원 품질이 낮아집니다. 무료 Token 신청을 권장하는 이유이며, 자세한 내용은 [설치 및 구성](/ko/installation.md)을 참고하세요.

## 재구성 → 자체 점검 → 페이지 내부 수정

시각적으로 비슷하다는 것이 편집 가능하다는 뜻은 아니며, 한 번의 생성으로 기준을 충족하기도 어렵습니다. 따라서 각 페이지의 재구성 담당자(로컬 모드의 메인 agent 또는 page worker)는 먼저 재구성한 다음 원본 이미지와 비교해 자체 점검하고, 누락된 텍스트·위치 오류·에셋 누락을 발견하면 페이지 내부에서 수정합니다. 결과가 원본에 충분히 가까워질 때까지 여러 번 반복할 수 있습니다.

이 사이클은 이 skill이 token을 많이 사용하는 근본적인 이유입니다. page worker가 한 페이지를 여러 번 시도할 수 있으며, 이미지 PPT를 편집 가능한 PPT로 변환하는 비용은 이미지 PPT 생성 비용의 2~3배가 될 수 있습니다. 하지만 편집할 수 없는 “픽셀 수준 복제본”을 전달하는 것은 의미가 없다고 봅니다. 같은 이유로 **편집 가능성이 꼭 필요하지 않다면 이 skill을 사용하지 않는 것을 권장합니다.**

## 다중 페이지 병렬 처리와 조회 가능한 상태

다중 페이지 입력은 메인 agent가 동시 실행 슬롯에 따라 page worker에게 병렬로 분배합니다. 각 페이지에는 독립 작업 디렉터리와 manifest가 있으며, 분배·페이지 결과·승인 상태는 모두 `editppt` CLI로 기록됩니다. 마지막에는 `editppt run finalize`가 페이지 순서대로 조립하고 deck 검증을 실행합니다. 자세한 과정은 [표준 워크플로](/ko/workflow.md)를 참고하세요.

변환 품질을 판단할 때 미리보기 이미지만 보지 마세요. PPTX 구조, 텍스트 커버리지, 에셋 출처, 미리보기/diff를 함께 확인해야 합니다.

## 두 skill의 역할 분담

이 skill은 “재구성”만 담당하고 “창작”은 담당하지 않습니다. 글, 보고서, 개요에서 새로운 PPT를 만드는 일은 [codex-ppt-skill](https://github.com/ningzimu/codex-ppt-skill)의 역할입니다. 두 skill의 역할은 다음과 같습니다.

- **codex-ppt**: 콘텐츠 → 이미지 기반 PPT. 시각적 일관성과 제어 가능한 흐름을 제공하며 대부분의 발표와 보고에 적합합니다.
- **image-to-editable-ppt**: 이미지 기반 페이지 → 편집 가능한 PPT. 객체별 편집이 실제로 필요한 경우에만 사용합니다.

두 skill의 자세한 비교는 [skill_duo_intro.pdf](https://github.com/ningzimu/image-to-editable-ppt-skill/blob/main/assets/skill_duo_intro.pdf)를 참고하세요. 이 PPT 자체도 codex-ppt skill로 생성되었습니다.
